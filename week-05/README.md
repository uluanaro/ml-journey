# SQL + Feature Engineering: Titanic

Извлечение данных из SQLite-запросами и подготовка признаков для модели.

## SQL-анализ

Данные загружены в SQLite, проанализированы запросами.

**Выживаемость по классу.** Первый класс — 63%, третий — 24% (разрыв в 2.6 раза). При этом в третьем классе больше всего пассажиров (491 против 216 в первом): низкие шансы касались самой многочисленной группы.

**Возраст и цена по классу и полу.** Пассажиры третьего класса заметно моложе (21–26 лет) первого (34–41). Женщины в каждом классе моложе мужчин.

**Оконная функция.** Для каждого пассажира выведена его цена билета рядом со средней по классу (`AVG(fare) OVER (PARTITION BY pclass)`) — позволяет находить тех, кто заплатил заметно выше или ниже среднего по своему классу.

## Feature Engineering

Созданы признаки для модели:

- **`family_size`** = `sibsp` + `parch` + 1 — размер семьи на борту.
- **`is_alone`** — плыл ли пассажир один. Проверено группировкой: семейные выживали примерно на 20 п.п. чаще одиночек (~50% против ~30%), вероятно, семьи держались вместе при эвакуации.
- **One-hot** для `sex` и `embarked` — категории без порядка нельзя нумеровать, иначе модель припишет им ложную иерархию.
- **`age_bin`** — возраст разбит на группы (ребёнок / подросток / взрослый / пожилой): для выживания важна возрастная категория, а не точная цифра.
